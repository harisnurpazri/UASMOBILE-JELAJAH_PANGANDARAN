**Alur Aplikasi (User Flow)**

1. Pengguna membuka aplikasi (`lib/main.dart` men-boot aplikasi).
2. Halaman utama menampilkan daftar destinasi (`lib/screens/home_screen.dart`).
3. Pengguna mengetuk item untuk membuka halaman detail (`lib/screens/detail_screen.dart`).
4. Di halaman detail, pengguna bisa melihat foto, deskripsi, lokasi di peta, dan menandai favorit.
5. Favorit disimpan secara lokal (shared_preferences / lokal DB) atau dikirim ke server (jika endpoint tersedia).

**Arsitektur & Struktur Folder**

Struktur umum di `lib/` dan peran tiap folder:

- `lib/main.dart`: Entrypoint aplikasi.
- `lib/config/`: Konfigurasi aplikasi (API base URL, keys, env).
- `lib/models/`: Model data (e.g., `place.dart`, `user.dart`).
- `lib/providers/`: State management dengan Provider (mis. `places_provider.dart`).
- `lib/services/`: Layanan seperti API client (`api_service.dart`), lokasi, dan penyimpanan lokal.
- `lib/screens/`: Semua layar UI (Home, Detail, Favorites, Settings).
- `lib/widgets/`: Komponen UI reusable (mis. `wisata_card.dart`, header, tombol).
- `lib/utils/`: Utilitas dan helper (format tanggal, konversi, konstanta).

Contoh jalur penting lain di root proyek:

- `assets/` â€” gambar, ikon, JSON statis
- `fonts/` â€” font yang dipakai aplikasi
- `android/`, `ios/`, `web/`, `windows/` â€” konfigurasi platform
